WSL + VS Code 搭建本地 Python 开发环境：从入门到避坑 正文 对于人工智能专业的本科生而言，搭建一套稳定、高效的本地开发环境是开启编程学习的第一步，而 WSL（Windows Subsystem for Linux）+ VS Code 的组合，正是兼顾 Windows 易用性与 Linux 开发便利性的最优解之一。WSL 允许用户在 Windows 系统中直接运行完整的 Linux 发行版（如 Ubuntu），无需双系统或虚拟机，既保留了 Windows 日常办公、娱乐的便捷性，又能获得 Linux 环境下原生的开发工具链，完美适配 Python、C++、ROS2 等人工智能相关技术的学习需求。VS Code 则凭借丰富的插件生态、强大的调试功能与 WSL 深度集成，让开发者可以在熟悉的图形界面中完成代码编写、环境管理、调试运行等全流程操作，大幅降低了 Linux 环境的使用门槛。 搭建这套环境的核心流程并不复杂：首先在 Windows 中启用 WSL 功能并安装 Ubuntu 发行版，随后在 VS Code 中安装 WSL、Python、Pylance 等核心插件，即可完成基础环境的配置。但在实际操作中，新手最容易踩的坑集中在路径适配、解释器选择、文件存储三个方面：WSL 与 Windows 的文件系统相互隔离，Windows 盘符在 WSL 中以 /mnt/盘符/ 的形式挂载，直接使用 Windows 路径会导致「文件不存在」错误；VS Code 需手动选择 WSL 环境下的 Python 解释器，否则会默认使用 Windows 环境的 Python，引发环境不匹配问题；为避免 C 盘空间占用，建议将 WSL 子系统、代码文件统一迁移至 D 盘，同时养成「代码存 D 盘、环境归 WSL」的习惯，从根源上杜绝路径错误与空间爆满的问题。 这套环境的优势不仅在于搭建便捷，更在于其可扩展性：无论是运行 Python 深度学习代码、调试 C++ 程序，还是部署 ROS2 机器人系统，WSL + VS Code 都能提供原生 Linux 环境的运行体验，同时支持 GPU 加速（适配 NVIDIA CUDA），满足人工智能专业从基础学习到项目开发的全场景需求。对于本科生而言，掌握这套环境的搭建与使用，不仅能解决日常编程作业的环境问题，更能提前适应工业界主流的 Linux 开发流程，为后续考研、求职打下坚实的基础。 标注信息 • 所用模型：豆包（Doubao）。
